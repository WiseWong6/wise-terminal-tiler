# OpenClaw UI 可视化优化 - 原始 PRD / 需求文档

## 核心目标
打造一个具备“赛博朋克”高科技感、实时性强、信息透明的 Agent 运行监控大屏。

## 关键功能点

### 1. 角色交互 (Agent Character)
- **视觉反馈**：小人头上需根据状态切换 Emoji（接收、思考、说话、闲置）。
- **实时气泡**：小人头上显示最新的流式对话，单条展示，具有驻留感。

### 2. 用户输入 (User Input)
- **布局**：用户输入区沉淀在页面最下方。
- **关联**：通过视觉线条（电线/管道）连接输入区与上方的消息通道（TUI, Feishu）。

### 3. 日志面板 (History Log)
- **左右对齐**：仿聊天室风格，左侧展示 Agent 动作，右侧展示用户输入。
- **信息完整性**：必须支持全量数据的“展开（Expand）”功能，不能被截断。
- **清洗逻辑**：剥离原本系统中冗余的机器元数据（如 `Sender untrusted metadata`）。

## 技术实现方案
- **监听机制**：采用双轨制。SSE 监听实时系统事件（动效触发），API 轮询监听全量对话历史（确保文字不截断）。
- **UI 框架**：Vanilla CSS 配合 HTML5，使用 HSL 色调实现发光动效。
- **动效**：基于 CSS Animations 实现呼吸灯、浮动小人、发光导线。
